## 犯罪者在「換現」策略上的常見選擇（高層次描述）
網路犯罪（特別是金融詐騙與勒索）普遍依賴「資產變現」與「壓迫受害者付款」。越直接的變現方式賺得越快，但也越容易被偵測與追蹤；相對地，犯罪者會在匿名性與可行性之間做權衡。
* 犯罪者會在「速度／匿名／風險」三者間做權衡：越快變現通常越易暴露；越匿名的方式（例如某些加密資產渠道）可能需要更多技術或中介，且也面臨被追蹤或被執法破解的風險。
* 各國與平台對資金流向的監控日益進步，使得傳統「快速換現」路徑的可行性降低，這也是犯罪團伙不斷變換手法與組織形式的原因之一。

### 1. 金融詐騙（Financial fraud）

金融詐騙常見目的是直接從受害者或企業帳戶中取得金錢。常見的高層次模式包括：利用被盜的支付資料（信用卡、銀行登入憑證）、發動帳號接管（account takeover），或是假冒/社交工程使受害者自行轉帳。這類方式能最直接把非法所得變現，因此在犯罪經濟中具有高度吸引力。

## 垃圾郵件

「雖然從個人層面來看，惡意垃圾郵件的數量似乎微不足道，但據估計，在2013年，全球每天約有1830億封電子郵件被發送與接收，因此惡意通信的數量可能非常可觀。根據Symantec（2013年）的估計，每天大約有3000萬封垃圾郵件被發送，而其中相當大的比例包含惡意軟體。這種數量龐大的垃圾郵件並不令人驚訝，因為據估算，若要讓一則垃圾郵件廣告達到盈利，每25,000位收件人中只要有一人打開郵件並在地下市場中進行購買即可（Symantec 2008）。2010年，僅是來自假藥品銷售的垃圾郵件，就為操作者帶來了270萬美元（約350萬澳元）的利潤（Krebs 2012），而全球網際網路服務提供商（ISPs）與用戶因垃圾郵件所付出的代價，則高達數十億美元（Anderson 等人，2013年）。一項關於垃圾郵件經濟學的最新研究（Rao 與 Reiley，2012年）估算，垃圾郵件發送者每年可能獲得約2億美元（約2.62億澳元）的總營收，而全球每年為了抵禦垃圾郵件所投入的花費，則高達200億美元（約260億澳元）。」**

### 2. 勒索 / 敲詐（Ransom / Extortion）

勒索通常透過兩種主要路徑：一是用惡意軟體（例如把資料加密）阻斷受害者對資產或系統的存取，要求以金錢換回解密；二是以威脅公開敏感資料（例如商業機密或個資）來逼迫受害者支付。勒索的核心在於對受害者造成（實際或聲稱的）不可接受損害，迫使其支付以避免損失或名譽受損。

* **缺點 / 風險**

  * **高可追蹤性**：銀行交易、商家監控與 KYC（了解你的客戶）機制使得大量現金化操作容易留下線索。
  * **暴露身份**：實體提領或收貨、反覆使用同一支付渠道、或與可識別的帳戶互動，都會增加被埋伏或被執法追蹤的機會。
  * **更重刑責與執法關注**：直接換錢被抓到後，證據通常較直接（交易記錄、物流紀錄），遭到偵辦的風險與量刑可能較高。
三、勒索/敲詐與金融詐騙的重疊與差異

* **重疊點**：兩者都以金錢為目的，經常會互相配合（例如先入侵竊取資料，再以勒索或出售資料變現）。某些犯罪團伙同時從事資料販售、詐欺詐領與勒索。
* **差異**：金融詐騙重在直接從金融系統榨取金錢；勒索則利用恐嚇或阻斷以換取付款，常伴隨較高的「談判」與心理戰成分。

## 二、間接變現 主要類型與高層次說明
此種方式也能直接變現，但由於沒有直接跟受害者對街，受害者可能不會察覺 受害者可能在無意識中蒙受重大損失
間接變現指犯罪者不是直接向受害者索款或當場竊取錢財，而是取得可轉售或可持續產生價值的資產（如個資、企業內部資料、智慧財產、受害裝置的算力），再透過第三方市場、長期濫用或其他管道換現。因為受害者往往短期內不會察覺，損失可能長時間累積或延後爆發。
* 受害者常於短期內不察覺，或發現時已造成累積性損害。
### 1. 智慧財產（IP）竊取

**模式概述**

* 盜取設計圖、原始程式碼、研發資料、專利相關文件或未公開的商業策略。
* 這些資料可能直接賣給競爭對手、仿製生產者，或用來啟動新詐騙（例如使用企業內部流程發動針對性攻擊）。

### 2. 個資兜售／黑市（Data theft & resale）

**模式概述**

* 竊取個人資料、支付資訊、帳號密碼、醫療紀錄或企業存取憑證，然後在暗網/地下市場分級販售。
* 買家再用這些資料進行銀行詐騙、身份盜用、社交工程或販售給其他惡意團體。

### 3. 暗中挖礦（cryptomining / cryptojacking）

**模式概述**

* 犯罪者在受害設備（伺服器、個人電腦、IoT 裝置等）上執行挖礦軟體，利用受害者的 CPU/GPU 計算資源為自己挖取加密貨幣。
* 常見特徵是裝置性能下降、耗電增加、散熱/硬體損耗加速。

* **加密貨幣挖礦** — 在受害設備上暗中執行挖礦軟體，消耗 CPU/GPU 電力等資源以挖掘加密貨幣，導致裝置變慢、耗電與硬體壽命縮短。過往瞄準高性能，像是服務器主機，榨乾資源；現在轉成控制大量設備，慢慢的低調的大量挖。[^6] [^7]

### 🔧 What's MaaS (Malware-as-a-Service)?

**Malware-as-a-Service** is a model where skilled hackers **create and sell or rent** malware tools to less skilled users (even script kiddies). It's similar to Software-as-a-Service (SaaS), but for cybercrime.

* **Buyers** don’t need technical skills.
* **Sellers** offer ready-to-use ransomware, phishing kits, trojans, etc.
* Includes dashboards, support, and tutorials.
* Payments are usually in cryptocurrency (like Bitcoin).

* **流量操作** — 殭屍網路能夠大量且自動化地將流量導向特定網站、影片或線上服務，用於人為提高瀏覽量、點擊數或觀看時長（例如刷流量、提升排名、操控推薦演算法）。

* **垃圾郵件與釣魚郵件散發** — 大量散發垃圾郵件或釣魚郵件，誘騙收件人點擊惡意連結、下載附件或洩露機密資訊；有時會用特定工具（如所稱之 Reactor Mailer）自動化發送。

* **散布更多惡意軟體或勒索程式** — 利用已入侵的設備散布更多惡意軟體或勒索程式，擴大感染範圍或加密受害者資料以索取贖金。

* **代理流量以隱藏攻擊者身分** — 將攻擊流量轉發或代理出去以隱藏真實來源，讓追蹤和阻斷攻擊者變得更加困難。
* **撞庫和暴力破解** — 利用大量已洩露的帳密組合自動嘗試登入（credential stuffing）或用暴力猜密碼（brute-force），以取得未授權存取。
* **進階攻擊** — 被用來掃描並利用已知或零日漏洞支援更大規模的攻擊活動，或作為「跳板」在企業或家庭網路內進行橫向移動。
